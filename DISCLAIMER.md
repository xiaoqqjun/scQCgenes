# scQCgenes 使用声明与免责说明

## 📢 重要声明

### 本包的用途

**scQCgenes** 是一个用于单细胞RNA测序数据分析的辅助工具，专门用于：

✅ 识别和过滤质控相关基因（线粒体、核糖体、预测基因等）  
✅ 简化数据分析流程，提高工作效率  
✅ 统一质控标准，便于结果重现  

### 本包不能用于

❌ **评估测序质量** - 过滤基因数量不代表测序好坏  
❌ **判断实验成败** - 请使用常规QC指标  
❌ **作为投诉依据** - 这不科学也不合理

## 🔬 关于Gm/Rik基因的科学解释

### 为什么会有这么多Gm/Rik基因？

1. **这是小鼠基因组的特点**
   - 小鼠基因组包含大量预测基因（predicted genes）
   - Gm = Gene model，基因组注释过程中的预测基因
   - Rik = RIKEN cDNA，日本理化学研究所发现的cDNA序列

2. **检测到Gm/Rik基因是正常的**
   - 单细胞测序捕获到这些转录本是预期现象
   - 不代表测序质量差
   - 不代表实验失败

3. **为什么要过滤？**
   - 这些基因缺乏功能注释
   - 难以进行GO/KEGG富集分析
   - 影响结果的生物学解释
   - **过滤是为了更好地分析，不是因为它们是"坏基因"**

### 真实案例

> **场景：** 某用户的数据中，20,000个基因里有6,000+个Gm基因（约30%）
> 
> **是否正常？** ✅ 是的，这在小鼠数据中很常见
> 
> **测序有问题吗？** ❌ 不一定，需要看其他QC指标
> 
> **该怎么办？** 
> - 检查常规QC指标（见下文）
> - 根据研究目的决定是否过滤这些基因
> - 不要单纯因为Gm基因多就认为测序有问题

## 📊 如何正确评估测序质量

### 应该看的指标

1. **测序深度（Sequencing Depth）**
   - Reads per cell
   - 推荐：>20,000 reads/cell（取决于实验设计）

2. **比对率（Mapping Rate）**
   - 多少reads能比对到基因组
   - 推荐：>70%

3. **细胞捕获效率**
   - 捕获到的细胞数
   - 是否达到预期

4. **每个细胞的基因数（nFeature）**
   - 检测到的基因数量
   - 推荐：500-6000（视细胞类型而定）

5. **每个细胞的UMI数（nCount）**
   - 总分子数
   - 反映测序深度

6. **线粒体比例（percent.mt）**
   - 线粒体基因表达占比
   - 高比例可能表示细胞死亡
   - 推荐：<10-20%

7. **Doublet比例**
   - 双细胞的比例
   - 推荐：<5%

### 不应该看的"伪指标"

❌ Gm基因的绝对数量  
❌ Rik基因的绝对数量  
❌ scQCgenes过滤掉的基因数

## 💬 常见误解澄清

### 误解1："过滤的基因越多，测序质量越差"
**真相：** 不是！小鼠基因组本身就有很多预测基因，过滤多少取决于：
- 基因组注释的完善程度
- 细胞类型表达的基因组成
- 你的分析目标

### 误解2："应该把过滤的基因数告诉测序公司"
**真相：** 没必要，也不科学。应该关注：
- 是否达到预期的测序深度
- 比对率是否正常
- 细胞质量是否合格

### 误解3："用了这个包，我的数据就干净了"
**真相：** 本包只过滤特定类型的基因，不能替代：
- 死细胞过滤
- Doublet去除
- Batch effect矫正
- 其他常规QC步骤

## 🎯 正确使用本包的原则

1. **明确你的研究目标**
   - 研究细胞周期？别删cell_cycle基因
   - 研究线粒体功能？别删mito基因
   - 常规分析？可以删Rik/Gm基因

2. **保留原始数据**
   ```r
   # 永远先保存原始数据
   saveRDS(seurat_obj, "original_data.rds")
   ```

3. **记录分析过程**
   ```r
   # 记录过滤了什么
   result <- filter_seurat_genes(obj, return_removed_genes = TRUE)
   write.csv(result$removed_genes, "removed_genes.csv")
   ```

4. **在论文中如实报告**
   ```
   "我们使用scQCgenes包过滤了线粒体基因、核糖体基因和
   预测基因（Rik/Gm），以便于后续的功能注释分析。"
   ```
